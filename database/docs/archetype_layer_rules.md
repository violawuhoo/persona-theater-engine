# Archetype Layer Rules

The active path is:

- `seed -> archetype json -> persona json`

## Archetype purpose

`archetype json` is the canonical mother-model layer.

It owns:

- core model logic
- stable style profile
- authored asset pools
- canonical prohibitions
- quadrant envelope compiled from authored seed quadrants

## Canonical owners

- `model_core`
  - owns `core_essence`
  - owns `social_stance`
  - owns `interaction_logic`
  - owns `emotional_logic`
  - owns `power_logic`
- `style_profile`
  - owns stable behavior and language summaries
- `assets`
  - owns slogan
  - owns signature line pool
  - owns reaction pattern pool
- `constraints.forbidden_drift`
  - owns prohibitions

## Anti-duplication rule

The archetype layer should not repeat the same seed meaning across multiple descriptive containers.

Avoid using archetype as:

- a display layer
- a docs-like expansion layer
- a runtime scene script
- a semantic mirror of the same seed concept under several names
